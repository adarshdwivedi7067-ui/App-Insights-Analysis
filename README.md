. About the Project
This project analyzes Google Play Store apps data to uncover insights about installs, ratings, reviews, and category distribution. The dashboard was built using Looker Studio and supported by SQL/Excel preprocessing.

2. Requirements
- Dataset: Google Play Store dataset
- Tools: Looker Studio, SQL, Excel, Python (optional)
- Deliverables: Dashboard, case study, insights, recommendations
3. Tools & Technologies
- Looker Studio – Dashboard creation
- SQL – Data extraction and cleaning
- Excel – Preprocessing and quick analysis
- Python (Pandas, Matplotlib, Seaborn) – Optional deeper analysis
4. Challenges Faced
- Large dataset with billions of installs → required cleaning and normalization
- Category imbalance (Games vs niche categories) → visualization needed scaling
- Ensuring dashboards remained clear and not overloaded with metrics
5. Insights
- 92.6% apps are free, driving maximum installs
- Games dominate installs (Subway Surfers, Candy Crush, 8 Ball Pool)
- Sports apps have strong user bases but lower installs compared to games
- Higher ratings correlate with higher installs and reviews
6. Recommendations
- Developers: Focus on free apps with monetization strategies
- Marketing: Prioritize high‑install categories like Games and Communication
- Product Teams: Improve ratings via UX updates and user feedback
- Business Strategy: Explore niche categories (Medical, Finance, Lifestyle) for growth
7. Outcome
- Built analytical mindset for app performance evaluation
- Delivered actionable insights for developers, marketers, and product managers
- Demonstrated ability to use BI tools for real‑world datasets
